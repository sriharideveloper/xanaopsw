SYSTEM OVERVIEW

Your Own Intelligence Command Center

XANA OS is a self-hosted, fully offline-capable intelligence dashboard. Think of it as your personal ops room — real aircraft overhead, ships at sea, satellites in orbit, global news, cyber threat feeds, and an AI that actually remembers your conversations. All on your machine. All free. None of it talks to anyone's cloud.

The AI memory system is the weird part. It doesn't fine-tune any model on your data. Instead, when you ask something, it pulls the most relevant chunks from your conversation history and drops them right into the prompt. The model reasons over your actual past — it just never had to be trained on it.

No GPU required. No API keys. No data leaves your machine. Swap models in one line.

RETRIEVAL ARCHITECTURE

How the Memory Works

No training. No fine-tuning. No mysterious black box that ate your data and mutated. Just retrieval — fast, transparent, and yours to inspect.

Q
YOUR QUESTION

You ask something. Could be a vague memory, a technical question, an investigation target — doesn't matter.

⟳
VECTOR SEARCH · ChromaDB

Your question is embedded and compared against tens of thousands of semantic vectors built from your conversation history. Top N most relevant chunks are retrieved.

⊕
PROMPT ASSEMBLY

System prompt + memory chunks + recent chat history + your question — all assembled into one rich context block.

AI
LOCAL LLM · Ollama

The model reasons over your data, grounded and specific. Runs entirely on your machine. Your conversation history is never uploaded anywhere.

DEPLOYMENT

Getting It Running

Two dependencies. Three commands. You're done.

01

Prerequisites

You need Python 3.10+ and Ollama — the local LLM runtime. Grab it at ollama.com and pull a model.

BASH ollama pull llama3.2 # default · ~4 GB · runs on any modern CPU ollama pull mistral # alternative ollama pull qwen2.5 # lighter option
02

Clone the Repository

BASH git clone https://github.com/sriharideveloper/xanaops.git cd xana-os
03

One-Line Setup & Launch

BASH ./setup.sh # creates venv, installs deps, checks Ollama ./start.sh # launches the app at localhost:8501

GLOBE, OSINT, and all live-feed modules work immediately — zero setup. The 3D globe is live the moment the page loads.

04

Build Your Memory Database (Optional · Highly Recommended)

This is the part that makes ORACLE, CHRONOS, NEURAL MAP, and DOSSIER actually interesting. Export your ChatGPT conversations and run the pipeline:

PIPELINE # 1. Export from ChatGPT → Settings → Data Controls → Export Data # 2. Copy conversations-*.json files into project root, then: python legacy/parse_chats.py # → master_chats.csv python legacy/1_prep_data.py # → paired_chats.csv python legacy/build-brain.py # → xana_memory_db/ (ChromaDB vectors) ./start.sh # restart — all memory modules now online

The first run downloads the embedding model (~90 MB). After that it's instant. CPU-only is totally fine — this isn't training, just inference.

PRIVACY NOTE

Your Data Never Leaves Your Machine

XANA OS runs entirely local. The LLM runs via Ollama — no API calls, no telemetry, no model vendor seeing your queries. Your conversation history and vector database live in your filesystem. The only outbound traffic is read-only requests to the public data sources listed above — no personal information is ever sent.

The only exception is the optional Windy webcams API, which requires a free key from api.windy.com. Even then, all you're sending is a location coordinate.
